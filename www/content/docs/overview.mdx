## Stack

### Recommended extensions (for VSCode)

- [UnoCSS](https://marketplace.visualstudio.com/items?itemName=antfu.unocss) - UnoCSS autocomplete
- [ESLint](https://marketplace.visualstudio.com/items?itemName=dbaeumer.vscode-eslint) - JavaScript/TypeScript linter

### Build tool

This template is centered around [Vite](https://vite.dev/) as it's very flexible and hackable, providing good DX while simultaneously handling client and server.

#### Why Vite?

- Way more performant with [rolldown](https://rolldown.rs/) (replacement for esbuild and rollup) than bundlers like webpack
- Very customizable when it comes to complex apps
- Versatile for stuff like PWA, different frameworks, in our case a seperate SMTP server and customizable build outputs
- Rich of plugins making it easy to extend the app

### Frontend

The frontend is built with [React](https://react.dev/), but it shouldn't be difficult to switch to another one you prefer.

#### Routing

Routing is made with [vite-plugin-rsc-pages](https://github.com/mrozio13pl/vite-plugin-rsc-pages) and [@vite/plugin-rsc](https://github.com/vitejs/vite-plugin-react/tree/main/packages/plugin-rsc) which is configured to mimic Next.js-like experience.

#### Styling

CSS is mostly done with [TailwindCSS](https://tailwindcss.com/).

#### UI Components

- [Radix UI](https://radix-ui.com/) - Accessible, unstyled component primitives
- [Lucide](https://lucide.dev/) - Community-backed SVG icon library

### Backend

The backend is build with [Hono](https://hono.dev/).

#### Why Hono?

- Easy to understand and integrate with Vite
- Very customizable and extendable
- Type-safe and has RPC client out of the box
